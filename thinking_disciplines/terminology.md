## Discipline Tiers

**Core disciplines** (the cognitive cycle): S → I → C
Sensemaking, Innovation, Critique. Operate on one question to produce one answer. Understand, generate, evaluate. The cycle IS the thinking.

**Boundary disciplines** (between cycles): R + N
Reflection, Navigation. Operate at the edge between one cognitive cycle and the next.
- R looks backward — how did the process perform?
- N looks forward — what are all possible next directions?

Core disciplines process. Boundary disciplines steer and learn.

1. self improvement competance 


     trusting threshold for the cognitive loop. if it is low we cant auto accept self changes..   
     What IS this threshold? It's the point where the system's CRITIQUE of its own changes is more reliable than random. If the system  
  proposes a change to sensemaking and then critiques that change, the critique must be good enough to catch bad changes and pass    
  good ones. Below this threshold: the system accepts changes that degrade it (false positives in self-critique). Above: the system  
  reliably improves.                                                                                                                 
   
  This is the self-improvement viability threshold — the cognitive competence level at which self-modification produces net positive 
  change. Below it, human gatekeeping is required. Above it, the loop can run autonomously.

  What this produces: The concept of a measurable threshold for autonomous self-improvement. The telemetry from self-improvement runs
   could MEASURE this: "of the last 10 self-proposed changes, how many actually improved the discipline when tested?" If the hit rate
   is >70%, the system is above threshold. Below 70%, human gatekeeping stays.    


    The Graduated Autonomy Model                                                                                                  

  Not binary (human gatekeeps vs. full autonomy). A measured gradient:                                                               
                                          
  Level 0: Human reviews every self-modification                                                                                     
  Level 1: Human reviews modifications that change discipline PROCESS (not just content)                                             
  Level 2: Human reviews only modifications flagged as UNCERTAIN by the system
  Level 3: Human sets strategic direction; system handles all tactical self-improvement                                              
  Level 4: System identifies its own strategic gaps and proposes directions to human   


    Progression is measured by self-improvement hit rate: % of self-proposed changes that actually improve telemetry in subsequent     
  runs. As hit rate climbs, autonomy level increases. If hit rate drops, autonomy level decreases.

  Regression Detection as the First Step                                                                                          
                                                                                                                                     
  Before autonomy can increase, the system must detect its own degradation. Build: cross-run telemetry tracking that compares each   
  discipline's metrics against its rolling average. If a self-modification causes a drop, flag it automatically.
