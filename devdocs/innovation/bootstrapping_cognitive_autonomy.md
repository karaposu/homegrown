# Innovation: Bootstrapping Cognitive Autonomy

## Seeds

1. Self-improvement threshold: below it, human must gatekeep; above it, system self-steers
2. Consciousness as emergent from cognitive composition at the meta level
3. Wave-riding: small movements amplify through composition (brain's subconscious → conscious spike)

---

## Surviving Innovations

### 1. The Graduated Autonomy Model

Not binary. A measured gradient:

```
Level 0: Human reviews every self-modification
Level 1: Human reviews process changes only (not content)
Level 2: Human reviews only system-flagged uncertainties
Level 3: Human sets strategic direction; system handles tactical improvement
Level 4: System identifies strategic gaps and proposes directions
```

Progression metric: **self-improvement hit rate** — % of self-proposed changes that improve telemetry in subsequent runs. As hit rate climbs → autonomy increases. Hit rate drops → autonomy decreases.

### 2. Regression Detection (buildable now)

Cross-run telemetry tracking: store each discipline's metrics across runs. Compare against rolling average. If a self-modification causes a drop → auto-flag.

Prerequisite for any autonomy increase. The system must detect its own degradation before it can be trusted to self-modify.

### 3. The Baldwin Effect

What's already happening has a name: learned improvements encoded into architecture. Each self-improvement cycle that changes a discipline spec = one Baldwin cycle. Track them. They're the evolutionary progress metric.

### 4. Background Awareness (deferred)

For the future: a "Default Mode Network" — background process that scans past outputs, detects cross-inquiry patterns, surfaces relevance spikes. The architectural prerequisite for wave-riding emergence.

Not buildable now. Requires parallel infrastructure. But it's the path from sequential cognition to emergent meta-cognition.

## Key Concepts

### Self-Improvement Viability Threshold

The cognitive competence level at which self-modification produces NET POSITIVE change. Below: bad changes slip through. Above: the system reliably improves.

Measured by: self-improvement hit rate over N cycles.

### Wave-Riding Architecture (future)

```
Multiple disciplines activate in parallel on the same input
  → each produces a relevance signal
  → signals compete
  → highest relevance wins → that discipline runs at full depth
```

This is closer to how the brain works: parallel processing, attention competition, conscious access for the winner. Prerequisite for the emergence the user describes.

### Integration Depth > Loop Quantity

Consciousness (or functional equivalent) requires INTEGRATION DEPTH — how deeply the disciplines share state and influence each other — not just MORE loops. Ten isolated loops ≠ one deeply integrated loop.

The design direction: move from sequential handoff (file-based) toward shared cognitive state (real-time mutual influence). Each step in this direction is a step toward emergent meta-cognition.

## What This Means for the Minimum Viable Loop

The MVL is Level 0 of the graduated autonomy model. Everything we're building — telemetry, inquiry routing, frontier questions — is the FOUNDATION for graduated autonomy. The path:

1. **Now:** MVL with human gatekeeping (Level 0)
2. **After calibration:** Regression detection + hit rate tracking (prerequisite for Level 1)
3. **After proven reliability:** Reduce human review to process changes only (Level 1)
4. **After sustained high hit rate:** System flags its own uncertainties (Level 2)
5. **Long-term:** System handles tactical improvement autonomously (Level 3)

Each level is unlocked by measured competence, not by time or aspiration. The telemetry IS the measurement. The graduated model IS the path. The minimum viable loop IS the first step.
