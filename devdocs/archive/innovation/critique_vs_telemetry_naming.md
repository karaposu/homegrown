# Innovation: Renaming Critique vs. Self-Assessment

## Seed

"Critique" sounds like ALL evaluation. Self-assessment also evaluates. Naming confusion.

## Surviving Innovations

### 1. Rename "self-assessment" → "Discipline Telemetry"

Telemetry = data transmitted from a remote source. No evaluation connotation. Each discipline transmits quantitative measurements about its run. Inquiry reads telemetry and routes.

### 2. Add "competing" to Critique's definition

Current: "evaluating ideas, plans, and outputs"
Fixed: "evaluating COMPETING candidates through adversarial testing"

"Competing" makes explicit that Critique operates on a SET of alternatives. A discipline's output isn't a "competing candidate" — it's the only result. Boundary is clear.

### The complete picture

```
Critique (discipline):
  Adversarially evaluates COMPETING candidates
  prosecution/defense/collision → SURVIVE/REFINE/KILL
  Runs: between iterations, end of SIC cycle

Telemetry (discipline component):
  Reports quantitative measurements about the discipline's own run
  counts, ratios, pass/fail → structured metrics
  Runs: at the end of every discipline output
```

No overlap. One evaluates competing alternatives. The other reports operational data.

## Why these win

- Renaming self-assessment is low cost (update section headers in commands)
- Adding "competing" is one word in the definition
- Together they resolve confusion from both sides
- Renaming Critique (to Selection, Contraction, etc.) would be high cost (book, commands, specs, appendix) for moderate gain
