# How to Run the SIC Loop for One Query

## The answer: S → I → C, sequential. Each step feeds the next.

```
/sense-making "the question"     → produces understanding
/innovate [sensemaking output]   → produces options (grounded in understanding)
/td-critique [S + I outputs]     → produces verdicts (which option wins)
```

## Why sequential (not parallel, not independent)

I DEPENDS on S's output (needs the understood situation as seed).
C DEPENDS on S's output (evaluation criteria) + I's output (candidates).
There's no way to run them independently or in parallel without each missing what it needs.

## Resolution scaling (not always full pipeline)

| Stakes | What to run |
|---|---|
| Routine ("next small step?") | /sense-making alone, or /wayfinding |
| Important ("which approach?") | S → C (candidates already known) |
| Major ("what to focus for a month?") | Full S → I → C |
| Diagnostic ("why isn't this working?") | S → Comprehend |

## Use /inquiry to manage sequencing

```
/inquiry "question" → CONFIGURE: S→I→C → "Next: /sense-making ..."
/sense-making ...
/inquiry [folder] → reads telemetry → "Next: /innovate ..."
/innovate ...
/inquiry [folder] → reads telemetry → "Next: /td-critique ..."
/td-critique ...
/inquiry [folder] → iteration complete → verdict
```

The human reviews between each step. Can redirect if S or I went wrong.

## This IS the MVL

Three commands. Three review points. Feed-forward. Discipline specs as configurations. Inquiry as sequencer. Human steering between steps.
