# Structural Sensemaking: How to Include Frontier Questions in Comprehend

## SV6 — Stabilized Model

**Frontier questions integrate as a named output of the CV progression, not a new component.**

They're what each phase leaves behind — questions the comprehender can now formulate but hasn't answered. The concept already exists implicitly; naming it makes it explicit.

**Key insight from the user:** Only generate UNANSWERED questions. Answered Q&A pairs are deducible from the CV body text. Frontier questions are genuinely new information — they exist nowhere in the body because the body only describes what WAS done, not what WASN'T.

**Integration approach:** One subsection ("Frontier Questions") after Comprehension Versions, plus one bullet per phase's "Produces" list. Minimal, non-disturbing, strengthens cross-session resume and coverage assessment.

**Three roles frontier questions serve:**
1. Direction — tells next person/session where to point effort
2. Boundary — maps edge of current model (no false confidence)
3. Depth signal — question EXISTS because comprehension got deep enough to see it

**Discipline spec = frontier questions only. Command can optionally also format tested predictions as Q&A pairs — that's a presentation concern, not a discipline definition.**
