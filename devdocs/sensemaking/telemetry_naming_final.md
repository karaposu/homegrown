# Sensemaking: Is "Run Metrics" Better Than "Telemetry"?

## SV6 — Keep Telemetry. The problem was using it alone without domain context.

"Telemetry" is the right LAYER name. "Run Metrics" would trigger software performance associations and lose the "transmitted for consumption" quality.

### The pattern

The layer is generic: **Telemetry**
The header is specific: **[Domain Name] (Telemetry)**

| Discipline | Header |
|---|---|
| Sensemaking | Saturation Indicators (Telemetry) ← already done |
| Innovation | Mechanism Coverage (Telemetry) |
| Critique | Convergence Telemetry |
| Comprehend | embedded in depth tests + final summary |

This matches how other output layers work: Transform is generic, but each discipline's transform has its own name (SV progression, CV progression, fitness landscape).
