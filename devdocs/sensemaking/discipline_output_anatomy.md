# Sensemaking: Telemetry and the Universal Output Anatomy

## SV6 — Stabilized Model

### Yes, rename to Telemetry

Telemetry = quantitative measurements about the discipline's run. No evaluation connotation. Dissolves the confusion with Critique.

### Telemetry is one of four universal output layers

Every discipline's runtime output has four layers:

| Layer | What it is | Discipline-specific? | Currently in all disciplines? |
|---|---|---|---|
| **Transform** | Core cognitive output | Yes | Yes (7/7) |
| **Progression** | Versioned snapshots of how work evolved | Yes | Yes (7/7) |
| **Telemetry** | Quantitative measurements of run quality | No — generic | Nearly (6/7) |
| **Frontier** | Coverage gaps expressed as questions | No — generic | Partial (2/7) |

### Two universal standards

**Spec anatomy** (what a discipline IS — from `what_are_they.md`):
1. Definition / philosophy
2. Structural components
3. Process model
4. Failure modes
5. Coverage strategy

**Output anatomy** (what a discipline PRODUCES — new):
1. Transform — core cognitive work
2. Progression — versioned evolution
3. Telemetry — operational metrics
4. Frontier — unanswered questions for further work

### Why "output layers" not "components"

"Components" already means discipline-specific internal parts (anchors, mechanisms, dimensions). "Layers" conveys they stack in the output. No naming collision.

- Rename "Self-Assessment" → "Telemetry" in all commands
- Add Frontier sections to the 5 disciplines that don't have them
- Document the output anatomy alongside the spec anatomy in `what_are_they.md`
