## Seed

Navigation is a boundary discipline. Two-tier system: core (S→I→C) + boundary (R+N). Innovation question: what emerges from the two-tier system for R/N interaction, N's discipline completeness, and the autonomous loop?

---

## Key Innovations

### 1. R feeds N (2a)

R runs FIRST at the boundary. N reads R's output. Learn first, then navigate.

Flow: SIC → R (process observations) → N (navigation map informed by R) → human selects → next SIC

If R detects "S was thin," N includes RE-RUN DEEPER with guideline "address the weakness R identified." R's findings become N's inputs.

**Convergence: 3 mechanisms (Combination, Extrapolation, Domain Transfer)**

### 2. N's failure modes (5a)

Five named failure modes:
- **Premature filtering** — showing only obvious options, missing subtle ones
- **Recency bias** — over-weighting C's output, under-weighting context-directed types
- **Action bias** — only content-directed types, missing process-directed
- **Enumeration without reasoning** — bare directions without WHY and guidelines
- **Scope fixation** — all directions within current scope, missing REFRAME/WIDEN

### 3. N's telemetry (5b)

Four telemetry points:
- **Type coverage** — how many of 15 types considered?
- **Category balance** — distributed across content/process/context, or clustered?
- **Guideline depth** — items have guidelines with WHY, or bare directions?
- **Excluded reasoning** — excluded section explains rejections?

### 4. N as `/navigate` command (4a)

Single command, one output (navigation map). Clean. Independently invocable.

### 5. OODA mapping (6b — REFINE)

R = Observe. N = Orient. Human = Decide. SIC = Act. The boundary pair IS the O-O of OODA. Elegant mapping — needs its own inquiry to test structurally.

## Assembly

**R→N sequential flow + failure modes + telemetry + command = COMPLETE DISCIPLINE SPECIFICATION for Navigation.**

All discipline components identified:
- Definition: enumeration of all possible next directions from current cognitive state
- Output: navigation map (direction+WHY, guidelines+each-WHY, confidence, excluded)
- Taxonomy: 15 types, 3 categories
- Process: read SIC output + R observations → assign types → produce map
- Failure modes: 5 named
- Telemetry: 4 metrics
- Command: `/navigate`

N is ready to be specified as `thinking_disciplines/navigation.md`.
