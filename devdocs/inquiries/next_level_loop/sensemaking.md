# Sensemaking: What's the ONE Thing for Next Level?

## SV6 — Bake improvement capture into MVL, then run it.

### The gap
The loop exists. Running is possible. But running without CAPTURING what worked/didn't/should-change is a demo, not a self-improving system. The capture mechanism is the missing piece.

### The ONE thing (two concrete actions)

**Action 1:** Add improvement capture to MVL.md's TERMINATE section:
```
## Improvement Capture
1. What worked? (which part of S, I, or C produced value?)
2. What didn't? (where was output shallow, wrong, or unhelpful?)
3. Config change? (which spec to edit and how? or "none")
Append to devdocs/improvement_log.md with date + question.
```

**Action 2:** Run /MVL on a real problem. Any real problem. Not meta.

### Why capture matters
- Without capture: each run is isolated. No Baldwin cycles. No accumulated learning.
- With capture: run → capture → improve spec → next run is better. The self-improvement loop closes.
- The capture is prompted by MVL (embedded in the flow, not separate — separate things get forgotten).
