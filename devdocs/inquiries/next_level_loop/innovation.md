# Innovation: Improvement Capture Formulations

## Seed
How should the improvement capture work to maximize self-improvement data quality while being minimal enough that the user actually does it?

## Three Survivors

### 1. One-line + instant Baldwin (lightest + most actionable)
```
## Quick capture
This run: [one sentence — what worked, what didn't]
Config change? [proposed edit — apply now? y/n]
```
One sentence. If config change, apply immediately. Baldwin cycle in one interaction.

### 2. Keep/Drop/Add retro (proven format)
```
## Retro
Keep: [what to keep]
Drop: [what to stop/change]
Add: [what to add]
```
Three words, three answers. Proven from Agile.

### 3. Frontier classification (zero-friction supplement)
Tag frontier questions as "about the problem" vs. "about the discipline."
Discipline-weakness frontiers accumulate as improvement data automatically.

## For critique
All three are combinable. The question is: which is the PRIMARY format?
The lightest (Survivor 1) or the most structured (Survivor 2)?
