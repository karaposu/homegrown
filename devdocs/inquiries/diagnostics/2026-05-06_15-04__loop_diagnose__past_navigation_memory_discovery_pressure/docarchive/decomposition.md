# Decomposition: Loop Diagnose - Past Navigation Memory Discovery Pressure

## Input

`_branch.md`, `exploration.md`, and `sensemaking.md` from this inquiry.

## Step 1 - Perceive Coupling Topology

The whole problem:

```text
Diagnose how the prior early-stage robust Route Memory Review policy made
PastNavigationMemoryFile presence load-bearing without a discovery contract,
and decide whether the later maintained-index proposal was a reasonable bridge
or an overcorrection.
```

### Elements

- Prior robust-mode rule.
- `PastNavigationMemoryFile exists` trigger.
- Missing discovery/search/index/fallback semantics.
- Human index question.
- Later narrow `PastNavigationMemoryFile Index` proposal.
- Discovery versus interpretation boundary.
- Deterministic search alternative.
- Maintained index staleness risk.
- Stage attribution across E/S/D/I/C and CONCLUDE.
- Maintenance candidates and evaluation gates.

### Coupling Gradients

Strong coupling:

- Source-present trigger and discovery semantics: the trigger cannot operate safely without discovery.
- Discovery and false absence risk: if discovery is weak, no-source skips become unsafe.
- Maintained index and stale-state risk: an index helps visibility but creates update obligations.
- Discovery and review authority: if discovery stores route dispositions, it becomes duplicate Route Memory Review.

Moderate coupling:

- Prior branch scope and attribution: the prior question was not discovery, but its finding proposed operational intake behavior.
- Human explicitness preference and index recommendation: explicit artifact culture made the index attractive but does not prove it optimal.
- Search alternative and maintenance target: deterministic search may satisfy the same interface without maintained state.

Weak coupling:

- Exact storage path and this diagnosis: path matters for implementation, not for diagnosing the prior miss.
- Future automation tooling and v1 discovery interface: tooling can wait for evidence.

## Step 2 - Detect Boundaries Top-Down

   - What exactly did the prior loop leave under-specified?

3. **Index As Bridge Or Overcorrection**
   - Was the later index recommendation a reasonable response, an overcorrection, or both?

4. **Discovery / Review Boundary**
   - What belongs to candidate discovery versus Route Memory Review interpretation?

5. **Stage Attribution**
   - Which prior stages likely contributed, and at what confidence?

6. **Maintenance Candidate Space**

These boundaries cut at low coupling. The correction delta describes evidence, the missed interface identifies the failure, index classification evaluates the later response, discovery/review boundary prevents authority drift, attribution localizes, maintenance converts diagnosis into options, and verdict/gates decide actionability.
