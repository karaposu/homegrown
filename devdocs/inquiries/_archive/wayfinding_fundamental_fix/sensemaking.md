# Sensemaking: Root Cause of Wayfinding's Failures

## SV6 — The definition contradicts the core question.

### Root cause (three levels)

1. **Surface:** TODO fixation — recommending insignificant items
2. **Structural:** Three missing capabilities — goal awareness, impact assessment, path enumeration
3. **Foundational:** The definition scopes wayfinding to state awareness ("sense where the search stands") but the core question demands action evaluation ("what action would change the landscape MOST?"). The mechanism can't answer its own question.

### Why this explains ALL symptoms

| Symptom | Caused by |
|---|---|
| TODO fixation | No goal → defaults to "what's undone" |
| Wrong answer to own question | No action evaluation → can't compare impact |
| Missing significance | Significance requires a goal to assess against |
| Fake gates | No path enumeration → treats plan order as only sequence |

### Three missing capabilities

| Capability | What it does | Why wayfinding needs it |
|---|---|---|
| **Goal awareness** | Know what we're trying to achieve | Impact is meaningless without a goal to measure against |
| **Impact assessment** | For each possible action, how much does it advance the goal? | Prioritizes actions by significance, not by "is it undone?" |
| **Path enumeration** | See ALL reasonable next actions, not just plan items | Prevents tunnel vision on existing plans |

### The fix (structural)

Drop "not planning" identity constraint. Expand the layers:

- **Present layer** gains: Goal + Possible Actions
- **New evaluation step:** For each action → impact toward goal (HIGH/MEDIUM/LOW/NEGATIVE). Select move from highest-impact action.
- **Trend layer** gains: Goal Velocity (advancing toward goal, not just productive)
- **Unchanged:** Memory layer, six moves, reachability, gates, core question
