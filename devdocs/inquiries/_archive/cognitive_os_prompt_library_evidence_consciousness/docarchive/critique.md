## Phase 1: Fitness Landscape

### Viable Region

Positions that describe Homegrown as a layered system: prompt substrate, protocolized workflow, cognitive runtime prototype, and unvalidated scientific hypothesis. Viable answers must include a validation path and consciousness caveat.

### Boundary Region

Positions that use "cognitive operating system" without enough caveat. These can be rescued if the term is qualified as prototype, metaphor, or aspiration.

### Dead Region

Positions that say:

- it is only a prompt library;
- it is already a mature cognitive operating system;
- its structured reflection proves or unlocks consciousness;
- impressive output is sufficient empirical evidence.

### Unexplored Region

Formal comparison against existing cognitive architecture literature is unexplored in this inquiry. It is not necessary for the user's current question, but it would matter if the project later makes academic claims.

## Phase 2 and 3: Candidate Verdicts

### Candidate A: Layered Identity

**Prosecution:** Layered framing can become vague if it avoids choosing a primary identity. It may sound like hedging.

**Defense:** The project really is layered. A single label loses information: "prompt library" loses ambition, and "cognitive OS" loses current implementation limits.

**Collision:** Defense wins. The layered model is not evasive because each layer has a concrete referent: storage, process, architecture, and evidence status.

**Verdict:** SURVIVE.

**Constructive output:** Use this as the master answer structure.

### Candidate B: Qualified Cognitive OS

**Prosecution:** "Operating system" risks overclaiming because Homegrown does not enforce execution, scheduling, or validation as software.

**Defense:** The system defines cognitive primitives, execution sequences, state files, lifecycle protocols, telemetry, and failure checks. Those are OS-like at the cognitive-process layer.

**Collision:** Both sides are strong. The claim survives only with explicit qualification.

**Verdict:** REFINE.

**Constructive output:** Say "experimental cognitive operating system prototype" or "cognitive runtime ambition," then immediately explain what is missing.

### Candidate C: Claims Ledger

**Prosecution:** A claims ledger may sound bureaucratic and could slow creative development.

**Defense:** The project is already near the line where internal vocabulary can outrun evidence. A claims ledger prevents category errors and turns the scientific question into testable work.

**Collision:** Defense wins. The user explicitly asked "evidence for what claim exactly?", so claim separation is central, not optional.

**Verdict:** SURVIVE.

**Constructive output:** The final answer should list claim types and evidence types.

### Candidate D: Consciousness-Adjacent, Not Consciousness

**Prosecution:** This may understate the possibility that recurrent metacognitive architectures are relevant to consciousness under functionalist theories.

**Defense:** It preserves that possibility while rejecting the stronger unsupported conclusion. The current system lacks enough integration, autonomy, intrinsic motivation, and self-maintaining continuity to justify an unlock claim.

**Collision:** Defense wins. The candidate is careful without being dismissive.

**Verdict:** SURVIVE.

**Constructive output:** State what the answer depends on: theory of consciousness, implementation depth, persistence, embodiment, and evidence standard.

### Candidate E: Validation-Before-Expansion Roadmap

**Prosecution:** The project is still creatively forming; forcing evaluation too early could freeze promising concepts.

**Defense:** The project already has enough conceptual structure to test. More taxonomy without evaluation increases self-reference risk.

**Collision:** Defense wins, with a caveat: validation should start small, not become a large research bureaucracy.

**Verdict:** SURVIVE.

**Constructive output:** Recommend small baselines, ablations, blind review, telemetry calibration, and state/resume tests.

### Candidate F: "It Is Already a Cognitive Operating System"

**Prosecution:** This fails current-implementation accuracy. The control layer is not mechanically enforced; the missing structural checker is a concrete example of the gap.

**Defense:** The conceptual architecture is unusually OS-like for a prompt system.

**Collision:** Prosecution wins because the word "already" turns a direction into an achievement.

**Verdict:** KILL.

**Seed extracted:** What would make the OS claim true in the future? Answer: executable routing, typed state, validators, enforcement, failure recovery, and measured reliability.

### Candidate G: "It Is Just a Prompt Library"

**Prosecution:** This misses loop composition, inquiry state, artifacts, protocols, telemetry, and reflection/navigation.

**Defense:** The system is materially stored as prompt-like Markdown instructions.

**Collision:** Prosecution wins against "just." Defense survives only as a substrate-level observation.

**Verdict:** KILL.

**Seed extracted:** Use "prompt-backed" or "Markdown-native" as implementation-medium language, not identity language.

### Candidate H: "This Can Unlock Consciousness"

**Prosecution:** The claim is much stronger than the mechanism. Current artifacts organize reasoning episodes; they do not demonstrate subjective experience or an integrated self-maintaining mind.

**Defense:** The project does include memory-like artifacts, reflection, recurrent loops, and self-evaluation, which are relevant to some functional theories.

**Collision:** Prosecution wins against "unlock." Defense survives only as "consciousness-adjacent research relevance."

**Verdict:** KILL.

**Seed extracted:** A responsible research question would be: which consciousness-adjacent capacities can this architecture support, and under which theory would those capacities matter?
