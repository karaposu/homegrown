# Decomposition: Cognitive OS, Prompt Library, Evidence, and Consciousness

## 1. Coupling Map

### Major Clusters

1. **Artifact identity cluster**
   - What Homegrown currently is.
   - Whether "prompt library" is true.
   - Whether "protocol system" is a better label.
   - Coupling: strong, because these labels all describe current artifact form.

2. **Architecture ambition cluster**
   - Whether "experimental cognitive operating system" is valid.
   - What OS-like functions are present.
   - What runtime functions are missing.
   - Coupling: strong, because the OS claim depends on the gap between current form and intended control layer.

3. **Scientific validation cluster**
   - What empirical evidence would look like.
   - Evidence for which exact claims.
   - Which claims are testable now.
   - Coupling: strong, because evidence has no meaning without a named claim.

4. **Consciousness cluster**
   - Whether the approach can unlock consciousness.
   - Which theories of consciousness make it plausible or implausible.
   - What capacities are consciousness-adjacent but not consciousness.
   - Coupling: moderate to scientific validation, but separable because consciousness requires additional assumptions.

5. **Evolution and honest positioning cluster**
   - Where the project is trying to evolve.
   - What should be claimed now.
   - What should be built next.
   - Coupling: moderate across all clusters because positioning depends on the answer to each.

### Natural Boundaries

- **Boundary A:** Current implementation vs future ambition.
- **Boundary B:** Labeling question vs evidence question.
- **Boundary C:** Empirical task-performance claims vs consciousness claims.
- **Boundary D:** Internal coherence vs external validation.

### Bottom-Up Boundary Validation

- Prompt files and skill specs naturally group into artifact identity.
- MVL/MVL+, conclude, resume, and state files naturally group into architecture ambition.
- Telemetry and missing checks naturally group into scientific validation.
- Reflection, memory, recursion, and self-modeling analogues naturally group into consciousness relevance, but they do not collapse into consciousness.

The boundaries are coherent. The largest hidden coupling risk is between "better metacognition" and "consciousness"; the final answer must keep them separate.

## 2. Question Tree

### Q1: What is being built here?

Verification criteria:

- [ ] Names the current artifact form.
- [ ] Names the functional process layer.
- [ ] Names the longer-term architectural direction.
- [ ] Avoids reducing the system to only one label.

### Q2: Is it an experimental cognitive operating system?

Verification criteria:

- [ ] Explains why the label is useful.
- [ ] Explains why the label is not strictly true yet.
- [ ] Identifies which OS-like functions exist.
- [ ] Identifies which OS-like functions are missing.
- [ ] States the qualified version of the claim.

### Q3: Is it a prompt library?

Verification criteria:

- [ ] Explains why the label is true at the storage/substrate layer.
- [ ] Explains why the label is misleading if used alone.
- [ ] Gives a better layered description.

### Q4: What would empirical evidence look like?

Verification criteria:

- [ ] Separates different claims.
- [ ] Gives examples of measurements.
- [ ] Names baselines and ablations.
- [ ] Distinguishes output-quality evidence from telemetry, learning, autonomy, and consciousness evidence.

### Q5: Evidence for what claim exactly?

Verification criteria:

- [ ] Produces a claims table or equivalent.
- [ ] Identifies near-term testable claims.
- [ ] Identifies far-term speculative claims.
- [ ] Prevents evidence from one claim being misused to support another.

### Q6: Can this approach unlock consciousness?

Verification criteria:

- [ ] Explains why it might, under some theories.
- [ ] Explains why it might not, under other theories.
- [ ] Names what the answer depends on.
- [ ] Distinguishes consciousness-adjacent functions from consciousness itself.
- [ ] Gives an honest current verdict.

### Q7: Where is the project trying to evolve at this stage?

Verification criteria:

- [ ] Identifies current stage.
- [ ] Identifies likely next stage.
- [ ] Identifies healthy and unhealthy evolution paths.
- [ ] Gives a concise honest opinion.

## 3. Interface Map

### Q1 -> Q2

- **Flow:** Current artifact description constrains the OS claim.
- **Direction:** One-way.
- **Interface:** If Q1 says the system is mostly text protocols, Q2 must qualify "operating system" as prototype or ambition.

### Q1 -> Q3

- **Flow:** Current artifact form determines whether prompt-library language is acceptable.
- **Direction:** One-way.
- **Interface:** If the storage medium is prompt-like text, Q3 must admit the prompt-library substrate.

### Q2 -> Q4

- **Flow:** OS-like claims generate evidence requirements.
- **Direction:** One-way.
- **Interface:** The more the project claims runtime control, the more evidence must test enforcement, routing, and reliability.

### Q4 -> Q5

- **Flow:** Evidence design depends on explicit claims.
- **Direction:** One-way.
- **Interface:** Q5 must enumerate claims before Q4 can specify measurements.

### Q5 -> Q6

- **Flow:** The consciousness claim is one possible claim, but it requires separate standards.
- **Direction:** One-way.
- **Interface:** Evidence for better task work cannot be reused as evidence for subjective experience.

### Q6 -> Q7

- **Flow:** Consciousness uncertainty constrains honest positioning.
- **Direction:** One-way.
- **Interface:** Q7 must avoid making consciousness the central near-term promise.

### Q2, Q4, Q5 -> Q7

- **Flow:** The evolution path depends on whether the project prioritizes runtime implementation or validation.
- **Direction:** Many-to-one.
- **Interface:** The honest next stage should be validation plus light runtime enforcement, not more speculative claims.

## 4. Dependency Order

1. **Q1: What is being built?** Must come first; all labels depend on it.
2. **Q3: Prompt library?** Can be answered immediately after Q1.
3. **Q2: Cognitive OS?** Depends on Q1 and benefits from Q3.
4. **Q5: Evidence for what claim?** Must precede specific evidence design.
5. **Q4: What evidence would look like?** Depends on Q5.
6. **Q6: Consciousness?** Depends on Q5 so it does not borrow evidence from other claims.
7. **Q7: Evolution and honest opinion.** Comes last as synthesis.

Parallel work is possible between Q2 and Q3 after Q1, but the final answer should present them together as layered labels.
