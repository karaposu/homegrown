# Critique: Which Improvement Capture Format?

## Verdicts

| Candidate | Verdict |
|---|---|
| One-line + instant Baldwin | **SURVIVE** — primary format |
| Keep/Drop/Add retro | **KILL as primary** — optional expansion for significant runs |
| Frontier classification | **SURVIVE as supplement** — enriches data without friction |

## The selected format

**Primary (every run):**
```
This run: [one sentence]
Config change? [proposed edit — apply now? y/n]
```

**Optional (significant runs):**
```
Full retro? Keep/Drop/Add:
```

**Automatic (zero friction):**
Frontier questions classified as "about the problem" vs. "about the discipline."

## The answer to the inquiry

Add this capture to MVL.md's TERMINATE section. Then run /MVL on a real problem. The self-improvement loop closes when: run → capture → config change applied → next run is better.
