## Candidate Verdicts

### 1. Input shape as orchestration signal
**Prosecution:** No real taxonomy yet. Premature formalization encodes current human-level understanding.
**Defense:** 4-5 shapes are directionally correct, used informally. Make the taxonomy LIVING.
**Verdict: SURVIVE (caveat)** — taxonomy is LIVING, not fixed.

### 2. Problem shape revealed by discipline failure
**Prosecution:** Requires unified failure detection. "Thin SIC → wrong discipline" is specific heuristic.
**Defense:** Detection exists in telemetry + reflection. Noise at single-run becomes signal at aggregate (Baldwin pattern).
**Verdict: SURVIVE** — cleanest innovation. Detection exists; needs aggregation.

### 3. Parallel discipline application
**Prosecution:** Reconciliation non-trivial; different output types. Level 0-2 has no parallelism.
**Defense:** Frontier only. Name the direction so architecture isn't accidentally serialized.
**Verdict: SURVIVE (frontier)** — roadmap only, no current build.

### 4. Chain as `_state.md` relationship
**Prosecution:** Proliferation of relationship types. Overlaps with CONTINUES FROM.
**Defense:** Bidirectional navigation is valuable but can be lightweight — a SPAWNED list, not new formal type.
**Verdict: REFINE** — use SPAWNED list in `_state.md`, not new relationship type.

### 5. Unix philosophy framing
**Prosecution:** Metaphorical. Unix has structured interfaces; discipline outputs are unstructured markdown.
**Defense:** For DOCUMENTATION, not specification. Pedagogy.
**Verdict: SURVIVE (caveat)** — documentation frame only.

### 6. Selection reasoning as audit trail
**Prosecution:** Overhead. Trivial selections produce low-quality reasonings.
**Defense:** Make it CONDITIONAL — reasoning required only for non-trivial selections.
**Verdict: SURVIVE (caveat)** — conditional on non-trivial selections.

### 7. Usage telemetry
**Prosecution:** "Unused = signal" is ambiguous without cause.
**Defense:** Signal + investigation is the standard Baldwin pattern.
**Verdict: SURVIVE** — simple invocation counter.

### 8. Selection layer at Level 4+
**Prosecution:** Future components, not current innovation.
**Defense:** Road-mapping; shapes current architecture to support future.
**Verdict: SURVIVE (frontier)** — roadmap anchor.

### 9. Palette evolves via Baldwin
**Prosecution:** Split/merge needs evaluation criteria.
**Defense:** Same mechanism as any Baldwin cycle — telemetry-based evaluation.
**Verdict: SURVIVE** — natural extension.

### 10. Multi-step chain composition (REFINE confirmed)
Let patterns emerge before naming.
**Verdict: REFINE (confirmed)**

### 11. CREATION as frontier (REFINE confirmed)
Outside current thinking-disciplines scope. Needed eventually for execution.
**Verdict: REFINE (confirmed)**

```
LEVEL 0-2 ARCHITECTURE (buildable now):

THREE-TIER PALETTE:
- Core cycle:   S, I, C
- Boundary:     R, N
- Situational:  E, D, Comprehension

CHAINING:
- Disciplines chain serially based on input shape
- Bidirectional navigation via _state.md:
  - CONTINUES FROM (child → parent) — existing
  - SPAWNED list in parent's _state.md — new, lightweight

ORCHESTRATION (human performs):
- Living shape taxonomy (grows with encounters)
- Selection reasoning for non-trivial selections only
- Usage telemetry: invocation counter per discipline

LEARNING SIGNALS (for Level 3+):
- Mistake-based shape classification (thin output → wrong discipline)
- Selection reasonings → orchestration training data
- Usage patterns → palette evolution feedback

FRAMING (documentation only):
- Unix-for-thinking: tools / pipes / shell

FUTURE COMPONENTS (Level 3+, not built):
- Input-shape classifier, selection model, outcome tracker
- Parallel discipline execution + reconciliation

OUT OF CURRENT SCOPE:
- CREATION discipline (execution, not thinking)
- Multi-step chain composition patterns (let emerge)
```

Refinements applied:
- Shape taxonomy → living
- Chain relationships → lightweight SPAWNED list
- Selection reasoning → conditional
- Unix framing → documentation only
- Parallel + selection layer → frontier only

No candidates killed in critique (innovation's kills remain: unified primitive, 50-discipline speciation, immune metaphor).
