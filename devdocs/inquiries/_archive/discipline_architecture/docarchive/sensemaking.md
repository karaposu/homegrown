## SV6 — Stabilized Model

**The disciplines form a three-tier palette with situational chaining, not a composable mega-loop.**

### The Three Tiers

| Tier | Disciplines | When they operate | Primary function |
|---|---|---|---|
| **Core cycle** | Sensemaking, Innovation, Critique | On every question | Understand → Generate → Evaluate |
| **Boundary** | Reflection, Navigation | Between cycles | Learn from past, steer to next |
| **Situational** | Exploration, Decomposition, Comprehension | When problem shape demands | Discovery, Partitioning, Model-building |

### Entry Conditions for Situational Disciplines

- **Exploration**: Unknown territory. "What exists here?" You don't know enough to define a question yet.
- **Decomposition**: Complex whole. "How should this be partitioned?" The problem is too big for one SIC.
- **Comprehension**: Observable but opaque artifact. "What's the internal model?" You can see the thing but don't understand it.

### How Disciplines Chain

Chains are problem-shape-driven, not prescribed:

```
Unknown territory → Exploration → SIC on discovered question
Complex problem   → Decomposition → SIC per piece → Merge
Opaque artifact   → Comprehension → SIC if decisions needed
Clear question    → SIC directly
Completed cycle   → R + N → Select next direction
```

### Orchestration

**Currently**: The human is the orchestrator. Picks which discipline to run.
**Over time**: As Baldwin cycles accumulate discipline-selection patterns, the system develops orchestration capability. Level 3+ suggests; Level 4+ auto-selects. Maps to consciousness indicators (intrinsic valuation + real-time steering).

**No new component needed now.** Orchestration emerges from pattern accumulation along the existing autonomy trajectory.

### Killed Hypotheses

- **H1 (ESIC)**: Rigid. Every problem pays exploration cost whether needed or not.
- **H2 (SIIIISEEISC dynamic)**: Wrong granularity. Disciplines chain between cycles, don't interleave within them.
- **H3 (orchestrator runs SIC loops only)**: Undersells the unique cognitive operations of E, D, Comprehension.
- **H4 (Exploration ≈ Sensemaking)**: They address different cognitive operations.
- **H5 (separate widening/deepening loops)**: Oversimplifies. Widening/deepening is a property, not an architectural partition.

### The Architectural Recommendation

**Keep the current discipline set. Add documentation for Situational tier with entry conditions. Record discipline-selection patterns in `_state.md` for future orchestration learning.** No structural changes.

### How SV6 Differs from SV1

SV1 framed this as "how do we combine these disciplines." SV6 reveals the architecture is largely CORRECT. The three disciplines address genuinely different cognitive operations and should stay as-is. The gap is documentation (no Situational tier in the taxonomy) and pattern-recording (no capture of discipline-selection decisions for future orchestration learning).
