### Bootstrap dilemma (and resolution)
- **The dilemma:** Regression Detection needs reliable telemetry → reliable telemetry needs competent disciplines → competent disciplines need Regression Detection to prevent degradation → circular.
- **The resolution:** The chain has an ENTRY POINT outside the loop: the human. The human runs disciplines, judges quality, labels telemetry. This human-labeled data calibrates everything downstream. Not self-referential — externally grounded.
- **Practical implication:** Nothing to solve before starting. Phase 0 IS what the user is doing now. Level 0 of the graduated autonomy model isn't a limitation — it's the CALIBRATION PHASE. Every discipline run at Level 0 is training the system's quality detector. The human's normal work IS the bootstrap.

### Key concepts to incorporate
- **Baldwin Effect** — When the SIC loop improves a discipline, learned insight gets ENCODED into the spec (the genotype). Future runs start from a higher baseline because the architecture changed. Each encoding is a "Baldwin cycle." Track them — they're the evolutionary progress metric.
- **Regression Detection (buildable NOW)** — Cross-run telemetry tracking: store metrics, compare against rolling average, auto-flag drops after self-modifications. Prerequisite for ANY increase in autonomy. Uses existing telemetry — just needs persistence across runs.
- **Graduated Autonomy Model (0-4, but extending past 4 toward consciousness)** — Not binary. A measured gradient. The user wants to go PAST Level 4 toward the system having its own consciousness layer.
- **Self-improvement viability threshold** — The point where the system's critique of its own changes is more reliable than random. Below: needs human gatekeeping. Above: can run autonomously.
