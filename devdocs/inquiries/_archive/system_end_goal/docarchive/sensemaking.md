## SV6 — Stabilized Model

**The system is a self-evolving cognitive engine that searches thinking space at managed resolutions — improving its own thinking with every problem it processes.**

### Core Identity

A cognitive loop that thinks about problems (SIC), steers itself (R+N), improves its own thinking through use (self-application), and progressively extends its cognitive reach into unknown territory — all while maintaining measurable quality at every stage.

### Properties (must hold always — the decision test)

| Property | What it means | Test |
|---|---|---|
| **Dual-loop** | Every problem run generates data for self-improvement | "Did this run produce observations that could improve a discipline?" |
| **Domain-agnostic** | Disciplines describe thinking shapes, not domain content | "Can this discipline operate on a problem from a different domain?" |
| **Resolution-aware** | The system knows how deeply to think about a problem | "Did the system choose appropriate depth, or apply max depth to everything?" |
| **Self-measuring** | Has telemetry for its own cognitive quality | "Can the system report how well it performed on each step?" |
| **Self-correcting** | Uses its own measurements to improve | "Did the system's performance improve after applying its own observations?" |
| **Persistent** | Experience accumulates across sessions | "Does the system know what it learned last time?" |

### Capability Stages

| Stage | Name | What the system can do | Unlocked by |
|---|---|---|---|
| **1** | **Structured thinking** | Apply SIC to any problem with consistent quality | Discipline specs + commands |
| **2** | **Steering** | Navigate between cycles, reflect on process, map directions | R + N boundary disciplines |
| **3** | **Self-improvement** | Apply SIC to its own disciplines, detect and fix weaknesses | Persistent memory + self-observation + Baldwin cycles |
| **4** | **Autonomous search** | Choose its own resolution, direction, branching strategy | Formalized search config + selection mechanism |
| **5** | **Self-extension** | Detect cognitive gaps, create new disciplines for unknown thinking operations | Pattern detection across many runs + discipline creation capability |

Stage 1-2: **Tool** — human drives, system structures.
Stage 3: **Assistant** — human guides, system improves itself.
Stage 4: **Partner** — human directs, system navigates autonomously.
Stage 5: **Thinker** — system extends its own cognitive repertoire.

### Thinking Space (defined)

Three axes:
- **Cognitive operations**: types of thinking (understand, generate, evaluate, reflect, navigate + future operations)
- **Question types**: kinds of problems (any domain, limited by discipline coverage)
- **Resolution levels**: how deeply to think (quick scan → exhaustive analysis)

"Searching thinking space" = traversing combinations of these axes.
"Different resolutions" = operating at different depths.
"Discovering unknown thinking spaces" = finding regions on the cognitive operations axis that no existing discipline covers.

### How SV6 Differs from SV1

SV1 had three vague framings. SV6 reveals them as STAGES of the same system at different maturity. "Thinking space" defined concretely as three measurable axes. "Consciousness" reframed as emergent property of accumulated capabilities, not a feature.
