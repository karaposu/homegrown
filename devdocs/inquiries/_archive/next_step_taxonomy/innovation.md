## Key Innovations

### 1. Types EMERGE from C's output — no selection needed (1c + 3b + 7c)

C already produces: survivors (→DEEPEN/DEVELOP), refinements (→REFINE), kill seeds (→PURSUE SEED), frontier questions (→INVESTIGATE FRONTIER). Telemetry produces: flags (→RE-RUN DEEPER). Scope check produces: gaps (→WIDEN).

The types don't need SELECTION. They're PRODUCED by C. Each C output naturally maps to a type. The steering system doesn't choose — it READS types from C's output and formats them as a plan.

**Convergence: 3 mechanisms (Lens Shifting, Inversion, Extrapolation)**

### 2. R produces the plan — R IS navigation (7b + 2c)

R reads C's output + telemetry + process observations. R assigns types, detects dependencies, identifies critical path, applies branch limits, produces the PLAN.

SICR becomes: Sense → Innovate → Critique → **Navigate**

R was always navigation. We called it reflection. The taxonomy is R's vocabulary. The plan is R's primary output. **R replaces wayfinding.**

**Convergence: 2 mechanisms (Extrapolation, Combination)**

### 3. Automatable vs human-required split (4a)

| Automatable (machine-readable trigger) | Human-required (needs judgment) |
|---|---|
| DEEPEN (C: SURVIVE) | REFRAME (subjective: is framing wrong?) |
| REFINE (C: REFINE) | REVISIT (judgment: which past finding?) |
| PURSUE SEED (C: KILL + seed) | DIFFERENT APPROACH (judgment: which approach?) |
| INVESTIGATE FRONTIER (C: frontier) | CONSOLIDATE (judgment: which inquiries?) |
| RE-RUN DEEPER (telemetry < threshold) | |
| WIDEN (scope check = NO) | |
| TERMINATE (answer covers goal) | |
| UNBLOCK (blocker identified) | |
| TEST (finding stated as hypothesis) | |
| DEVELOP (survivor marked ready) | |
| MERGE (branches complete) | |

11 automatable, 4 human-required. This IS the graduated autonomy boundary.

### 4. The complete navigation cycle

```
S → I → C → R(Navigate):
  C produces: survivors, refinements, seeds, frontiers, telemetry
  R reads: C's output + process observations
  R produces: TYPED PLAN with dependencies
    → Types assigned from C's output (mostly automatic)
    → Dependencies from type relationships (mostly automatic)
    → Priority by marginal impact on goal
    → Branch limit applied (top N)
  Human approves plan → branches created → each runs SIC
```
