## Seed

Extended MVL = (E || D || S) → I → C with optional intra-I recursion. Orchestration IS the enhanced MVL. Cognitive-faculties analogy. Instrumental rigidity. Innovation question: what sharpens this further, what's missing, what better framings?

## Key Innovations

### 1. Explicit reconciliation step in I (5a)

Three streams (E, D, S) may contradict. Before synthesis, I must reconcile: does E's territory align with D's structure? With S's anchors? Where they clash, resolve (or flag as open). Reconciliation ≠ synthesis — it handles contradictions first.

Implementation: a reconciliation PRELUDE to I's existing process. Identify contradictions across E/D/S → resolve using strong anchors → then generate candidates from reconciled input.

**Convergence: 1 mechanism**, but structural necessity.

### 2. Ensemble methods framing (6a)

Multi-head IS an ensemble. E, D, S are diverse models; I aggregates.

Imports:
- **Voting / weighting**: I weights outputs by confidence
- **Stacking**: I as meta-model; recursion = retraining on prior meta-output
- **Diversity is a feature**: contradictions across streams aren't bugs, they're the ensemble's value

Grounds multi-head in proven ML patterns.

**Convergence: 1 mechanism**

### 3. Scientific method analogy (6b)

The extended MVL IS the scientific method structured as disciplines:
- E || D || S = observation / decomposition / hypothesis (parallel early phase)
- I = experimentation (generate testable candidates)
- C = evaluation
- R, N = reflect + reorient

Replaces both Unix-for-thinking AND cognitive-faculties-of-a-mind. More concrete — the scientific method IS cognitive faculty integration, proven over centuries.

**Convergence: 1 mechanism**

### 4. Self-improvement observations per run (4c)

Force every extended MVL run to produce observations about its own quality: did E help? Did D contribute? Was I recursion needed? Every run is dual-purpose — solve + self-observe. Accelerates Baldwin cycles.

**Convergence: 1 mechanism**

### 5. Stop-recursion criterion (5b)

Hard limit: max 3 recursions.
Soft criterion: stop when pass N produces < 20% new novel candidates vs N-1.
Telemetry-driven: stop when critique-hit-rate improves < 10% between passes.

Required for recursion to be safe.

**Convergence: 1 mechanism**

### 6. Telemetry-driven I recursion trigger (2b)

When should I recurse? When pass 1's critique KILLS > 70% of candidates, pass 2 runs with those kills as seeds. Tight feedback loop: critique signals failure → innovation retries with informed seeds.

**Convergence: 1 mechanism**

### 7. Backward compatibility (4a)

`/mvl` (classic SIC) and `/mvl-extended` (EDS+I+C) coexist. User picks. Preserves flexibility.

**Convergence: 1 mechanism**

### 8. First-time user wrapper (1a)

`/mvl-extended` = single command that runs E, D, S, I, C in sequence. Reduces invocation friction from 5 commands to 1.

**Convergence: 1 mechanism**

### 9. Data-driven palette refinement (7a + 7b)

After 50+ runs, patterns emerge: E non-trivial rate, D contribution rate, recursion hit rate. Use data to refine palette membership. If E's non-trivial rate < 20%, reconsider its Core status.

**Convergence: 2 mechanisms**

### 10. VOID capability (5c) — FRONTIER

System can decline MVL when problem is too trivial, too ambiguous, or already answered. At Level 0-2 human does this implicitly. At Level 3+ becomes explicit.

**Convergence: 1 mechanism**, frontier.

## Assembly

**The refined extended MVL:**

```
PHASE 1 — Multi-head observation (parallel; ensemble):
  E → exploration map
  D → decomposition tree  
  S → sensemaking anchors

PHASE 2 — Reconciliation + Synthesis:
  I reconciles contradictions across E/D/S
  I synthesizes, generates candidates
  OPTIONAL RECURSION: trigger when critique-kill-rate > 70%
  stop when max 3 OR new-novel rate < 20%

PHASE 3 — Evaluation:
  C evaluates

PHASE 4 — Boundary:
  R reflects (with self-improvement observations per run)
  N maps next directions

BACKWARD COMPATIBILITY:
  /mvl (classic) = S → I → C
  /mvl-extended = (E||D||S) → I(+recursion) → C

ANALOGY:
  The scientific method (observe/decompose/hypothesize →
  experiment → evaluate → iterate), structured as a pipeline.

FRONTIER (Level 3+):
  True parallel E||D||S with streaming input
  VOID capability (decline MVL)
  Three-separate-SIC-cycles mode for hardest problems
```
