# Structural Exploration: What Should Be Added to Sensemaking v2?

## Design Constraint

Sensemaking is a delicate operation. Additions must be LIGHTWEIGHT — awareness priming, not procedural checkpoints. Failure modes as RECOGNITION PATTERNS, not as steps to execute.

---

## 5 Additions (all lightweight)

### 1. Named Failure Modes

Recognition patterns, not checkpoints:

| Failure Mode | Feels like | Corrective |
|---|---|---|
| **Status Quo Bias** | Defending an established structure because it's documented | "Would I reach this conclusion if it were undocumented?" |
| **Premature Stabilization** | Model clicked fast, only 2-3 perspectives checked | Check: did fewer than 3 perspectives produce surprises? |
| **Anchor Dominance** | One anchor does all the work, all ambiguities resolve toward it | "If I removed this anchor, would the model collapse?" |
| **Perspective Blindness** | All perspectives agree, nothing produced discomfort | "Which perspective would be most uncomfortable? Check that one." |
| **Clean Resolution Trap** | Resolution is elegant but counter-argument not tested structurally | State WHY the counter fails structurally, not just by precedent |
| **Self-Reference Blindness** | Evaluating something that shares assumptions with your framework | Need external reference points — empirical evidence, cross-discipline comparison |

### 2. Coverage Strategy (Saturation Indicators)

Not gates — indicators that signal when to keep going:

- **Perspective saturation** — new perspectives stop producing new TYPES of anchors
- **Ambiguity resolution ratio** — below ~70% resolved = not done
- **SV delta** — if SV6 ≈ SV1, sensemaking was superficial
- **Anchor diversity** — anchors from multiple types + multiple perspectives

### 3. Internal Consistency Sub-Check

Add to Definitional/Consistency perspective: "Does the established definition contradict ITSELF? Does its stated purpose align with its actual mechanisms?"

### 4. Structural Grounds Requirement

Sharpen Ambiguity Collapse: "State WHY the counter-interpretation fails structurally. 'The spec says X' is precedent, not evidence. If you can only dismiss by citing precedent, the resolution is LOW CONFIDENCE."

### 5. Accommodation Trigger

When new perspectives keep destabilizing the current model — don't keep patching. Drop back to SV2-SV3 and re-extract anchors using the destabilizing perspectives as PRIMARY sources.

---

## What NOT to Add

| Rejected | Why | Risk |
|---|---|---|
| Adversarial prosecution at every ambiguity | Fragments the flow, turns sensemaking into mini-critique | HIGH rigidity |
| Prediction testing of SV output | Changes sensemaking's nature — that's Comprehend's job | HIGH rigidity |
| Mandatory perspective count | Artificial — saturation indicators are better | MEDIUM rigidity |
| Formal convergence criteria | Sensemaking convergence is inherently less measurable | MEDIUM rigidity |
